关联多个远程仓库

大类 命令 含义
连接到远端仓库
ssh-keygen -t rsa -C <youremailaddress> 在 ~/.ssh/ 里生成公钥
在 GitHub/Gitee 设置里添加公钥
git remote add <origin> git@gitee.com:<用户名>/<仓库名> 添加远端仓库源到本地仓库。ssh格式。
git remote rm <origin> 删除连接
从远端拉取代码到本地
git pull <远程主机名> <远程分支名>:<本地分支名>
git pull 简写
git pull origin master 如果远程分支是与当前分支合并
则冒号后面的部分可以省略
git pull origin master:brantest 将远程主机 origin 的 master 分支拉取过来
与本地的 brantest 分支合并
拷贝一个仓库到本地 git clone <url> git clone https://github.com/tianqixin/runoob-git-test
git clone https://github.com/tianqixin/runoob-git-test another-runoob-name 重命名为 another-ruboob-name
把本地代码推到远端
git push <远程主机名> <本地分支名>:<远程分支名>
git push 简写
git push <远程主机名> <本地分支名> 如果本地分支名与远程分支名相同
则可以省略冒号：
git push origin master 将本地的 master 分支推到远端的分支
更新被拒绝，因为当前分支的最新提交落后于其他分支
merge/rebase
git fetch origin 获取远程更新<br/>git merge origin/master 把更新的内容合并到本地分支
